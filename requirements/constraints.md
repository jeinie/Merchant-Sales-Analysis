# 가맹점 매출 분석 서비스 구현 예외사항 및 제약사항

## 1. 제외 기능 (구현하지 않음)

### 1.1 데이터 소스 연동
- 외부 데이터 수집 엔진 (크롤러 등) 직접 구현 (미정)
- 전일자 거래 데이터 배치 스크립트로 데이터베이스 insert
- 결제 승인 데이터 실시간 연동은 하지 않고 배치 처리

### 1.2 가맹점 관리 기능
- 신규 가맹점 등록 및 수정/삭제 기능 (데이터베이스 직접 입력을 가정)

### 2. 인증 및 보안
- 복잡한 사용자 인증 (OAuth, SNS 로그인)
- 다단계 인증 (2FA, OTP)
- 상세한 역할 기반 접근 제어 (RBAC) (관리자/영업사원 수준의 기본 구분만 구현)

### 1.4 알림 시스템
- 카카오 알림톡/SMS 발송
- 이메일 발송 자동화

---

## 2. 기술적 제약사항

### 2.1 지도 API
- 초기 구현 시 공공 데이터 또는 무료 범위 내의 지도 API(Google Maps, Kakao Maps, Naver Maps 등)를 활용하며, 고도화된 유료 레이어는 제외함.

### 2.2 AI 분석 범위
- AI는 제공된 매출 데이터와 비교 지표를 바탕으로 한 텍스트 기반 인사이트 도출에 집중함.
- 복잡한 외부 변수(날씨, 유동인구, 경기 지표 등)를 결합한 고도의 예측 모델링은 MVP 범위에서 제외함.

### 2.3 데이터 처리
- 대용량 로그성 데이터의 실시간 처리는 지양하며, 월별/일별 집계 데이터를 기준으로 분석을 수행함.

---

## 3. 기타 예외사항
- 다국어 지원 기능은 포함하지 않음 (한국어 전용)
- 사용자 매뉴얼 및 가이드 문서의 시스템 내 내장 기능 제외
- 실제 상용 수준의 부하 테스트 및 성능 최적화 (MVP 이후 단계)
